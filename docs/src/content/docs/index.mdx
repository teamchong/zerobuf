## What it does

JS and WASM share one `WebAssembly.Memory`. zerobuf defines a binary layout (tagged values, strings, arrays, objects) and provides accessor objects on the JS side that read/write directly to WASM linear memory.

```typescript
import { zerobuf } from "zerobuf";

const buf = zerobuf(wasm.exports.memory);
const point = buf.create({ x: 1.0, y: 2.0, label: "origin" });

point.x = 3.14;  // writes to WASM memory
transform(point.__zerobuf_ptr);  // WASM reads the same bytes
console.log(point.x);  // reads WASM's output
```

## Current state

- JS library with defineProperty accessors (V8-optimized reads, ~8M ops/sec)
- Zig library with matching binary layout and C ABI exports
- Dynamic objects: push, extend, grow, add properties
- All types: null, bool, i32, f64, bigint, string, bytes, array, object
- Not published to npm — internal use across *mode repos
